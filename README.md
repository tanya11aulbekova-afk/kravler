# KRAVLER — маркетплейс игровых товаров

Демо-сайт маркетплейса игровых товаров (аккаунты, валюта, ключи, скины, буст) в красно-чёрно-белом стиле.

Сайт полностью статический — без сборки и без сервера. Шрифты лежат локально, интернет для работы не нужен.

## Что внутри

| Файл | Что это |
|---|---|
| `index.html` | Главная: промо-баннеры, игры, популярные лоты |
| `catalog.html` | Каталог с фильтрами по игре и разделу, поиском и сортировкой |
| `product.html` | Страница товара с карточкой продавца и похожими лотами |
| `assets/` | Стили, скрипт с демо-данными, шрифты |

## Быстрая проверка (без GitHub)

Распакуйте архив и откройте `index.html` двойным кликом — сайт откроется в браузере. Всё кликабельно: фильтры, поиск, переходы между страницами.

## Как запустить через GitHub (получить публичную ссылку)

Понадобится 5–7 минут, ничего устанавливать не нужно.

**Шаг 1. Создайте аккаунт на GitHub**
Зайдите на [github.com](https://github.com) и зарегистрируйтесь (если аккаунт уже есть — просто войдите).

**Шаг 2. Создайте репозиторий**
1. Нажмите **+** в правом верхнем углу → **New repository**.
2. В поле *Repository name* впишите, например, `kravler`.
3. Выберите **Public** (обязательно, иначе бесплатный хостинг не заработает).
4. Нажмите **Create repository**.

**Шаг 3. Загрузите файлы сайта**
1. На странице нового репозитория нажмите ссылку **uploading an existing file**.
2. Распакуйте архив у себя на компьютере.
3. Перетащите в окно браузера **всё содержимое папки** (файлы `index.html`, `catalog.html`, `product.html` и папку `assets` целиком — важно тащить именно папку, чтобы структура сохранилась).
4. Дождитесь окончания загрузки и нажмите зелёную кнопку **Commit changes**.

**Шаг 4. Включите GitHub Pages**
1. В репозитории откройте вкладку **Settings** (шестерёнка).
2. В левом меню выберите **Pages**.
3. В блоке *Build and deployment* → *Branch* выберите **main**, папку оставьте **/ (root)** и нажмите **Save**.

**Шаг 5. Откройте сайт**
Через 1–2 минуты обновите страницу Settings → Pages — сверху появится ссылка вида:

```
https://ВАШ_НИК.github.io/kravler/
```

Это и есть публичный адрес сайта — его можно открывать с любого устройства и кидать кому угодно.

> Если по ссылке ошибка 404 — подождите ещё минуту-другую и обновите страницу. Если не помогло, проверьте, что `index.html` лежит в корне репозитория (а не внутри лишней папки).

## Примечания

- Кнопки «Купить», «Войти», «Продать» показывают уведомление «демо-версия» — оплата и авторизация не подключены, это макет.
- Товары — демо-данные, они лежат в `assets/app.js` (массив `OFFERS`), там же легко поменять названия и цены.
